Nginx管网络流量，Gateway管业务逻辑
流量管理：反向代理统一入口，负载均衡分发流量（轮询/权重/IP Hash），限流防打爆（每秒10请求/每IP 100连接）
性能优化：静态资源直接返回不走后端



1、nginx 反向代理 - 统一入口：Nginx收到请求后用proxy_pass转发给后端服务器，拿到响应后再返回给客户端。
2、负载均衡 - 流量分发
3. 限流 - 防止打爆
4. 静态资源分离 - 减轻压力


1、nginx 反向代理 - 统一入口
客户端只访问Nginx，Nginx转发到后端多个微服务，隐藏内部架构。
2、负载均衡 - 流量分发
轮询、权重、IP Hash 
轮询：默认
权重：weight
IP Hash ：IP Hash

3. 限流 - 防止打爆
每秒最多10个请求、每IP最多100个连接，超过限制直接返回503，保护后端服务。
4. 静态资源分离 - 减轻压力
图片、CSS、JS直接由Nginx返回，不走后端服务：
用 location 匹配静态文件后缀，直接从Nginx磁盘返回，不走后端服务。
